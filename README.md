# TaskFlow — Веб-трекер задач для IT-команд 🚀

**TaskFlow** — это легковесное многопользовательское веб-приложение для управления задачами (мини-аналог Trello или Jira). Проект разработан на языке Python с использованием фреймворка Flask и реляционной базы данных SQLite.

Проект создан в рамках подготовки к обучению в **ЙОТК** по направлению *«Информационные системы и программирование»*.

---

## 🛠️ Стек технологий

* **Backend:** Python 3, Flask (Маршрутизация, Сессии, Flash-сообщения)
* **Database:** SQLite 3 (Реляционные связи, Внешние ключи)
* **Frontend:** HTML5, CSS3 (Модульная верстка на Flexbox, Шаблонизатор Jinja2)
* **Валидация:** Модуль `re` (Регулярные выражения в Python)

---

## 🔑 Ключевые возможности и архитектура

1. **Реляционная база данных (SQLite):**
   * Таблица `users` для хранения аккаунтов.
   * Таблица `tasks` для хранения задач.
   * Организована связь **«один-ко-многим»** через внешний ключ (`FOREIGN KEY`), что позволяет разделять задачи между разными пользователями.
2. **Безопасность и валидация паролей:**
   * Встроенный модуль `re` проверяет пароль при регистрации на соответствие жестким правилам: длина от 8 символов, наличие минимум одной цифры, одной заглавной и одной строчной буквы.
   * Защита от **SQL-инъекций** за счет использования безопасных плейсхолдеров (`?`) во всех SQL-запросах.
3. **Управление сессиями (Session Management):**
   * Использование `session` для авторизации пользователей. Защита приватных страниц (`/dashboard`, `/add_task`) от неавторизованного доступа.
   * Автоматический вход в систему сразу после успешной регистрации.
4. **Интерфейс (UI/UX):**
   * Всплывающие уведомления об ошибках (`flash`) выводятся прямо внутри форм без перезагрузки на пустую страницу.
   * Современный адаптивный дизайн на Flexbox CSS в нейтральных тонах.
   * Полная реализация системы **CRUD** (Создание, Чтение и Удаление задач).

## 📁 Структура проекта

```text
taskflow/
│
├── app.py               # Главный файл бэкенда (логика, маршруты, сессии)
├── init_db.py           # Скрипт инициализации и создания таблиц БД
├── database.db          # Файл базы данных SQLite (генерируется автоматически)
│
├── static/              # Папка со статическими файлами сайта
│   └── css/
│       └── style.css    # Стили оформления интерфейса (Flexbox CSS)
│
└── templates/           # HTML-шаблоны для отображения в браузере (Jinja2)
    ├── login.html       # Страница авторизации (Вход)
    ├── register.html    # Страница регистрации нового пользователя
    └── dashboard.html   # Личный кабинет (Панель со списком задач)
```

## 🚀 Инструкция по запуску

### 1. Подготовка базы данных
Перед первым запуском необходимо создать таблицы в базе данных и добавить тестового пользователя. Для этого выполните скрипт в терминале:
```bash
python init_db.py
```
*После этого в корне проекта появится файл `database.db`, а в системе будет создан стандартный аккаунт администратора (`Логин: admin / Пароль: 1234`).*

### 2. Запуск веб-сервера
Запустите главный файл приложения:
```bash
python app.py
```

### 3. Проверка в браузере
После запуска сервер начнет работу локально. Откройте любой браузер и перейдите по адресу:
```text
http://127.0.0.1:5000
```
